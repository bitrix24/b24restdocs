# Delivery Requests for Shipping Services: Overview of Methods

{% note tip "" %}

If you are developing integrations for Bitrix24 using AI tools (Codex, Claude Code, Cursor), connect to the [MCP server](../../../../sdk/mcp.md) so that the assistant can utilize the official REST documentation.

{% endnote %}

Delivery requests synchronize an order in Bitrix24 with the statuses and messages from an external shipping service. For example, you can update the tracking number, change the delivery date, or send a notification to the customer.

This is necessary to keep the information about the cargo movement up to date and to inform the participants in the delivery process.

> Quick Navigation: [all methods](#all-methods)

## Data Structure of the Request

The method [sale.delivery.request.update](./sale-delivery-request-update.md) updates the request data through the parameters `STATUS`, `PROPERTIES`, and `FINALIZE`.

The `PROPERTIES` parameter transmits additional delivery data, while `STATUS` indicates the current state of the request. To completely replace the set of properties, use `OVERWRITE_PROPERTIES=Y`.

The method [sale.delivery.request.sendmessage](./sale-delivery-request-send-message.md) creates notifications for the request. Specify the recipient using the `ADDRESSEE` parameter, and the text, title, and status of the message in the [`MESSAGE`](./sale-delivery-request-send-message.md#parametr-message) object.

## Linking Delivery Requests with Other Objects

**Shipping Services.** A delivery request is linked to a specific shipping service by `DELIVERY_ID`. Manage services through the methods [sale.delivery.*](../delivery/index.md).

**Order and Shipments.** The request is associated with an order through shipments. The request identifier is generated by the external system in the scenario [Creating a Delivery Order](../webhooks/create-delivery-request.md).

## How to Work with Delivery Requests

1. Obtain the `DELIVERY_ID` of the shipping service using the method [sale.delivery.getlist](../delivery/sale-delivery-get-list.md).
2. Use the `REQUEST_ID` that the external system returns in the webhook [Creating a Delivery Order](../webhooks/create-delivery-request.md).
3. Update the status and properties of the request using the method [sale.delivery.request.update](./sale-delivery-request-update.md).
4. Send notifications to the manager or recipient using the method [sale.delivery.request.sendmessage](./sale-delivery-request-send-message.md).
5. Delete any outdated requests using the method [sale.delivery.request.delete](./sale-delivery-request-delete.md).
